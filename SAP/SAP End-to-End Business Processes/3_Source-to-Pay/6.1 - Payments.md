- All payments are overseen by the **Finance (FI) Department**. MM does not need to be involved
## Steps for Payment
- **Configure** payment method, bank, etc. This is usually *one-time* only
- **Select items** for payment
	- Vendor ID
	- Payment due date
	- Company code
	- Payment type
	- etc.
- **Calculate** payment amount - includes cash discounts
- **Post** payment document
	- This updates the ff:
		- *Vendor account*
		- *Subledger*
		- *Individual accounts* associated with items
## Payment Run
- A process configured to pay one or more invoices
- Can be tested before executing fully
### Characteristics of a Payment Run
- **Run Date**
- **Run ID**
- **Parameters**
	- *Payment items*
	- *Payment method*
	- *Payment date*
	- *Company code/s*
	- *Next payment run scheduled*
### Scheduling Payment Run
- Payment runs can be tested before being truly executed. This is called a **payment proposal**.
	- Once the proposal is run, a log is outputted for review and editing.
- After proposal, the payment run can be **scheduled** & executed. The medium must be configured.
	- DME file - EDI, DME, check print
