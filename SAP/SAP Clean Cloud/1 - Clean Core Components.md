**Core** - "dimensions" or "pillars" of an ERP systems
**Clean** = performing best practices

> **NOTE!** A newly provisioned SAP Cloud ERP system is clean per definition

> **NOTE!** The goal is to ***get*** and ***keep*** the core clean
### Core Components of an ERP System
- **Processes**
- **Extensions**
- **Data**
- **Integrations**
- **Operations**
### Clean Core Components
- **Minimalism**
- **Modularity** - loosely-coupled
- **Scalability**
- **Maintainability**
- **Stability and Reliability**
### Other Key Characteristics of a Clean Core System
- Simplified codebase
- Unneeded add-ons and third-party integrations are removed
- Involves not just code but also business processes, data, and operations
### Other Benefits
- Data synchronization is automatic, preventing "stale" (outdated) data or the need to manually update documents
- Updates are always automatic so IT doesn't need to implement upgrades or downtime
- Development and extensibility are both end-to-end and cloud-native, which meant that extensions and modifications are clean (if the system is clean core
	- **ABAP Cloud Development Model**
	- **SAP S/4HANA Cloud Extensibility Model**
- Centralized platform allows for improved coordination and reduced miscommunication
